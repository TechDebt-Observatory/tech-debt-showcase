# 6-Dimensional Documentation Framework

A comprehensive framework for documenting legacy code across six dimensions:

1. **Syntax** - Function signatures, parameters, return values
2. **Structure** - Code flow, algorithms, data structures
3. **Intent** - Purpose, design decisions, assumptions
4. **History** - Evolution, why changes were made
5. **Architecture** - Relationships with other components
6. **Technical Debt** - Known issues, workarounds, improvement opportunities

- `specification/` - Detailed framework specification
- `templates/` - Doxygen templates and examples
- `examples/` - Annotated code examples
